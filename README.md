## Описание

Репозиторий содержит клиент‑серверную игру **Крестики‑нолики** с общими игровыми моделями и двумя клиентами:

- **Backend** — модульная реализация сетевой логики игры (сервер и консольный клиент).
- **Frontend** — Android‑приложение с UI на Jetpack Compose и сборкой в двух вариантах: Client и Server.

Подробная документация проекта находится в файле [Documentation(RUS).pdf](Documentation(RUS).pdf).

## Структура проекта

- **KotlinProjectBack/backend** — сервер и консольный клиент, общая игровая логика:
  - **general** — общие сущности игры: `IGame`, `TicTacToeGame`, сетевые DTO.
  - **server** — сервер, ожидание подключений, обмен ходами, управление состоянием.
  - **client** — консольный клиент с поиском и подключением к серверу в сети.
- **KotlinProjectFront/app** — Android‑клиент/сервер с UI на Compose:
  - productFlavors: **client** и **server**
  - задачи Gradle `runClient` и `runServer` для быстрого запуска на устройстве.

## Основные возможности

- Сетевая игра в крестики‑нолики по локальной сети
- Сервер ожидает подключение и проводит игру по очереди ходов
- Клиент умеет сканировать сеть и выбирать доступный сервер
- Единая игровая логика и модель данных, используемая в обеих частях
- JSON‑обмен ходами и состояниями

## Технологический стек

- **Язык:** Kotlin
- **Сборка:** Gradle (Kotlin DSL)
- **Сеть:** Ktor Network + TLS
- **Сериализация:** kotlinx.serialization (JSON)
- **Асинхронность:** Kotlin Coroutines
- **Android:** Jetpack Compose, Android Gradle Plugin
- **Минимальная версия Android:** 24

## Быстрый старт

### Backend (консольный сервер/клиент)

Проект находится в `KotlinProjectBack/backend` и состоит из трех модулей: `server`, `client`, `general`.

### Frontend (Android‑приложение)

Проект находится в `KotlinProjectFront`. Доступны два flavor’а:
- `client` — клиент игры
- `server` — сервер игры

Запуск через Gradle задачи:
- `runClient`
- `runServer`

## Примечания

- Проект ориентирован на локальную сеть (LAN).
- Логика игры общая для серверной и клиентской частей.
